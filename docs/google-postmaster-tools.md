# Google Postmaster Tools 登録手順書(taf-design.com)

> 作成: 2026-07-22 / 対象: taf-design.com(エックスサーバー)
> 目的: **Gmailから見た自ドメインの「評価・迷惑メール率・認証成功率」を数値で監視**する
> 費用: **0円** / 作業時間: **10分**(+ DNS反映待ち最大24時間)
> 前提: タスク1(SPF/DKIM/DMARC設定)が済んでいること

---

## 0. これは何のツール?

- **Gmail(受信側)が、あなたのドメインをどう評価しているか**を教えてくれる無料の管理画面。
- 見られる主な数値:
  - **ドメインの評価(Domain Reputation)**: High / Medium / Low / Bad の4段階
  - **迷惑メール率(Spam rate)**: 受信者が「迷惑メール報告」した割合
  - **SPF / DKIM / DMARC の認証成功率**: タスク1の設定が実際に効いているかの答え合わせ
  - **暗号化(TLS)率** など
- タスク1で「設定した」→ タスク2で「Gmailに正しく認識されているか数値で確認できる」という関係。
  DMARCレポート(生ログ)より圧倒的に読みやすいダッシュボード。

> ⚠️ 大前提: **データが表示されるにはある程度の送信量が必要**(Googleの目安で1日あたり
> 概ね100通以上を継続送信するドメイン)。今のtaf-design.com(リード8件+CF7通知)では
> しばらく「データ不足」表示になる可能性が高い。**それでも今のうちに登録・ドメイン認証だけ
> 済ませておく**のが正解(MailPoet/Brevoで配信量が増えたら自動でデータが溜まり始める)。

---

## 1. 手順(6ステップ)

### STEP 1: ツールを開いてGoogleでログイン

- URL: **https://postmaster.google.com/**
- 普段使っているGoogleアカウント(Gmail)でログイン。
  → 屋号用のGoogleアカウントがあればそれで。個人用でも機能上は問題ないが、
    後で引き継ぎしやすいよう**事業用アカウントに統一**するのがおすすめ。

### STEP 2: ドメインを追加

- 画面右下の**赤い「＋」ボタン**(ドメインを追加)をクリック。
- ドメイン名を入力: **`taf-design.com`**
  - `https://` や `www.` は付けない。ドメインだけ。
- 「NEXT / 次へ」をクリック。

### STEP 3: 認証用のTXTレコードが表示される

- こういう文字列(DNS TXTレコード)が表示される:

  ```
  google-site-verification=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  ```

- この画面は**閉じずに残したまま**、別タブでエックスサーバーの作業へ(STEP 4)。
  値をコピーしておく。

### STEP 4: エックスサーバーのDNSにTXTレコードを追加

```
サーバーパネル(https://secure.xserver.ne.jp/xapanel/login/xserver/server/)
  └─ カテゴリ「ドメイン」内の【DNSレコード設定】
      └─ 「taf-design.com」の【選択する】
          └─ 「DNSレコード追加」タブ
              └─ 下記を入力 → 【確認画面へ進む】→【追加する】
```

| 項目 | 値 |
|---|---|
| ホスト名 | (空欄のまま = taf-design.com 自体) |
| 種別 | **TXT** |
| 内容 | `google-site-verification=xxxx...`(STEP 3でコピーした文字列そのまま) |
| 優先度 | 0(そのままでOK) |

> 📌 注意: これは**SPFのTXTとは別の行**として追加する(SPFは `v=spf1...`、
> こちらは `google-site-verification=...`)。TXTレコードは何行あってもOKなので、
> SPFを消したり上書きしたりしないこと。

### STEP 5: Postmaster側で「認証(VERIFY)」を押す

- STEP 3の画面に戻り、**「VERIFY / 確認」**ボタンをクリック。
- 「認証が完了しました」と出れば成功。
- **DNS反映前だと失敗する**。その場合は数時間〜最大24時間おいて再度「VERIFY」を押す
  (レコード自体が正しければ、時間が経てば必ず通る)。

### STEP 6: 完了 — あとはデータが溜まるのを待つ

- ダッシュボードにtaf-design.comが並ぶ。
- 最初は各グラフが「**データが不足しています(Not enough data)**」表示のはず。これは正常。
- 配信量が増えると、数日〜1週間で数値が入り始める。

## 2. 登録後、何を見ればいい?(配信を始めてから)

MailPoet/Brevoでメルマガを送り始めたら、週1回くらい覗いて以下をチェック:

| 指標 | 見方 | 目標 |
|---|---|---|
| **Spam rate(迷惑メール率)** | 一番大事。ここが上がると全体が届かなくなる | **0.1%未満**を死守(0.3%超えたら赤信号) |
| **Domain Reputation** | ドメインの信用度 | High または Medium をキープ |
| **Authentication(SPF/DKIM/DMARC)** | 各認証の成功率 | できるだけ100%に近く。低ければタスク1の設定に漏れがある |
| **IP Reputation** | 送信元IPの評価 | High(共有IPなので他社の影響も受ける) |

### Spam rateを上げないための基本

- **オプトイン(本人が登録した)リストにだけ送る** → タスク5(offer2のオプトインチェック)に直結
- **配信解除リンクを必ず入れる**(MailPoet標準装備)+ 見つけやすく
- 件名で釣らない・送りすぎない(月1〜2回ペースは健全)

## 3. つまづきポイント

| 症状 | 対処 |
|---|---|
| VERIFYが失敗する | DNS未反映。数時間おいて再挑戦。`nslookup -type=TXT taf-design.com` で `google-site-verification=` の行が見えるか確認 |
| いつまでもデータが出ない | 送信量不足。これは異常ではない。MailPoet/Brevo稼働後に自然に溜まる |
| 数値がガタガタ | 送信量が少ないと1通の影響が大きく出る。量が増えると安定する |
| ログインできない | Postmaster ToolsはGoogleアカウント必須。屋号用アカウントの有無を確認 |

---

## 4. 次のアクション

1. ✅ 今すぐ: STEP 1〜5でドメイン認証だけ済ませる(データは後から)
2. → **タスク3: MailPoet導入 + Brevo接続**(配信量を作る)
3. → 配信開始後、週1でSpam rate と Reputation を確認 → 悪化したら送り方を見直す
