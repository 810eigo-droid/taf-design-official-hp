# お問い合わせフォームの通知が届かないのは、フォームの故障ではなかった話

お問い合わせフォームを設置したあと、必ずやるのが送信テストです。

名前を入れて、メールアドレスを入れて、本文を書いて、送信。

画面には「送信しました」と出る。

でも、肝心のメールが届かない。

このとき、最初に疑うのはフォームの設定です。

フォームプラグインが悪いのか。
通知先メールアドレスを間違えたのか。
WordPress側で何か止まっているのか。

もちろん、その可能性もあります。

でも最近は、フォーム自体が動いていても、メールが受信側に信用されず、迷惑メールに入ったり、届きにくくなったりすることがあります。

原因は、メールの「本人確認」が整っていないことでした。

## メールにも本人確認がある

メールは、差出人を自由に書けてしまう仕組みです。

だから受信側は、

- このサーバーから送っていいメールなのか
- メールに付いている署名は本物なのか
- 認証に失敗したメールをどう扱えばいいのか

を見ています。

ここで出てくるのが、SPF・DKIM・DMARCです。

名前だけ見ると難しそうですが、役割はかなりシンプルです。

- SPF：このサーバーから送っていい、と宣言する
- DKIM：メールに電子署名を付ける
- DMARC：認証に失敗したメールの扱いを決める

お問い合わせフォームの通知や自動返信メールも、この認証の影響を受けます。

## 2024年以降、メールはより厳しく見られている

以前は、多少設定がゆるくてもメールが届いていたかもしれません。

でも今は、Gmailなどの受信サービスが、送信者の認証をかなり重視しています。

本人確認ができないメールは、迷惑メールに入ったり、そもそも届きにくくなったりします。

これは大きな会社のメルマガだけの話ではありません。

小さなWebサイトのお問い合わせフォームでも、Gmail宛に通知を飛ばすなら関係があります。

## エックスサーバーなら無料で設定できた

今回確認したのは、エックスサーバーでの設定です。

やったことは、大きく3つです。

1. DKIMをONにする
2. SPFレコードがあるか確認する
3. DMARCを「何もしない」から設定する

特にDKIMは、サーバーパネルからONにするだけで進められました。

SPFは、すでに入っている場合があります。ここで注意したいのは、SPFを2つ作らないことです。

SPFは1ドメインに1つだけ。

良かれと思って追加すると、逆に認証エラーになります。

DMARCも同じく、まずは厳しい設定にしないことが大事です。

最初は「何もしない（p=none）」で観測する。

いきなり拒否にすると、正しいメールまで止まる可能性があります。

## フォームは集客の最後の出口

お問い合わせフォームは、サイトの最後の出口です。

せっかく興味を持ってくれた人が問い合わせてくれたのに、通知が迷惑メールに埋もれていたら、とてももったいない。

フォームを作ることと、フォームから届くメールを整えることは、セットで考えた方がいいと感じました。

Web制作では、見た目や文章だけでなく、こういう裏側の設定も大事です。

## 今回の学び

- フォームの送信完了と、メールが届くことは別
- 届かない原因はフォーム故障とは限らない
- SPF・DKIM・DMARCは小さなサイトにも関係する
- SPFとDMARCは二重登録しない
- DKIMはエックスサーバーなら管理画面から設定しやすい
- フォーム制作とメール到達設定はセットで考える
