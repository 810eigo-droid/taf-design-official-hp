# お問い合わせフォームのメールを届きやすくするために確認したい3つの認証

お問い合わせフォームを作ったら、送信テストをします。

でも、画面では送信完了になっているのに、メールが届かないことがあります。

このとき、フォームの設定だけを見ていると原因にたどり着けないことがあります。

最近は、受信側が「このメールは本当にこのドメインから送られたものか」を厳しく見ているからです。

そこで確認したいのが、SPF・DKIM・DMARCです。

## 1. SPF

SPFは、「このサーバーからメールを送っていいです」と宣言する設定です。

エックスサーバーでは、最初から設定されていることもあります。

確認するときは、DNSレコードのTXTに `v=spf1` で始まる行があるかを見ます。

ここで注意したいのは、SPFを2つ作らないことです。

SPFは1ドメインに1つだけ。

メルマガ配信サービスなどを追加するときも、新しいSPFをもう1行足すのではなく、既存のSPFに追記する形になります。

## 2. DKIM

DKIMは、メールに電子署名を付ける設定です。

受信側はその署名を見て、メールが改ざんされていないかを確認します。

エックスサーバーでは、サーバーパネルからDKIMをONにできます。

ネームサーバーもエックスサーバーを使っていれば、必要なDNSレコードが自動で追加されるケースがあります。

フォーム通知や自動返信メールを安定させたいなら、まず確認したい設定です。

## 3. DMARC

DMARCは、SPFやDKIMの認証に失敗したメールをどう扱うかを決める設定です。

いきなり厳しい設定にする必要はありません。

最初は「何もしない（p=none）」で観測するのがおすすめです。

ここで大事なのは、DMARCも1ドメインに1つだけということ。

すでにDMARCがあるのに、別サービスの案内どおりにもう1つ追加すると、認証が壊れることがあります。

## 設定後はテストする

設定したら、実際にメールを送って確認します。

無料の採点サイト mail-tester を使うと、SPF・DKIM・DMARCが通っているかをチェックできます。

Gmailにテスト送信して、「メッセージのソースを表示」から確認する方法もあります。

SPF: PASS
DKIM: PASS
DMARC: PASS

このように表示されていれば、認証は通っています。

## まとめ

お問い合わせフォームのメールが届かないとき、フォームだけを見るのではなく、送信ドメイン認証も確認してみてください。

確認したいのはこの3つです。

- SPF：送っていいサーバーを宣言する
- DKIM：メールに電子署名を付ける
- DMARC：失敗したメールの扱いを決める

フォームは、問い合わせを受け取るための大切な出口です。

見た目だけでなく、届くところまで整えておくと安心です。
