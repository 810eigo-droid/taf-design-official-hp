# WordPressでメルマガを始めるなら、最初はMailPoet×Brevoで十分かもしれない

メルマガを始めたいと思ったとき、最初に迷うのが「どの配信サービスを使うか」です。

有名なサービスはたくさんあります。

でも、始めたばかりの小さなサイトで、いきなり月額費用をかけるのは少し重い。

そんなときに選択肢になるのが、**WordPressのMailPoetと、無料SMTPのBrevoを組み合わせる方法**です。

## MailPoet×Brevoは何がいいのか

一番のメリットは、WordPressの中でメルマガ運用を始められることです。

MailPoetを使うと、WordPress管理画面の中で、

- 購読フォームの作成
- 購読者リストの管理
- ニュースレターの作成
- ステップメールの設定

ができます。

そして、実際の送信部分はBrevoに任せます。

つまり、MailPoetが「作る・管理する」、Brevoが「届ける」という分担です。

## 最初から高機能サービスでなくてもいい

もちろん、本格的に何万人へ配信するなら、専用の有料配信サービスを検討した方がいいです。

でも、最初の段階ではそこまで必要ないことも多いです。

たとえば、

- お問い合わせ後のフォローメール
- 資料請求後のステップメール
- 既存のお客様へのお知らせ
- 月1回のニュースレター

このくらいなら、まずは無料枠で試してもいいと思います。

Brevoは無料プランで1日300通まで送信できます。小規模な運用なら、十分に検証できます。

## ただし「届く設定」は必要

気をつけたいのは、無料だからといって適当に設定していいわけではないことです。

今のメールは、送信者の本人確認がかなり厳しく見られています。

特に大事なのはこのあたりです。

- 送信者メールアドレスは独自ドメインにする
- SPF・DKIM・DMARCを整える
- Brevo側のドメイン認証を通す
- SMTP設定を正しく入れる
- 配信解除リンクを消さない

メルマガ本文を作る前に、まず「ちゃんと届く土台」を作る必要があります。

## SMTP設定で見落としやすいところ

MailPoetとBrevoをつなぐときは、SMTP設定を入れます。

ここで間違えやすいのが、ログイン情報とポート番号です。

ログイン欄には、自分のメールアドレスではなく、Brevoが発行する専用ログインを入れます。

また、環境によっては587番ポートで接続できないことがあります。

私の環境では、587では接続エラーになり、465 + SSL/TLSに変更したら通りました。

この情報は地味ですが、同じところで止まっている人にはかなり役に立つはずです。

## どんな人に向いているか

MailPoet×Brevoは、こんな人に向いています。

- WordPressサイトをすでに持っている
- まずは無料でメルマガを試したい
- 購読者がまだ少ない
- ステップメールを小さく始めたい
- Webサイトとメルマガを同じ管理画面で扱いたい

逆に、最初から大量配信をしたい人、細かい分析や高度なセグメント配信を重視する人は、有料の配信サービスも検討した方がいいです。

## まとめ

WordPressでメルマガを始めるなら、最初の選択肢としてMailPoet×Brevoはかなり現実的です。

ただし、大事なのは「無料でできる」ことだけではありません。

ちゃんと届くように、送信ドメイン認証やSMTP設定まで整えること。
