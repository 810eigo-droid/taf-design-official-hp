# メルマガ設定で587番ポートにハマった話。MailPoet×Brevoは465で通った

WordPressでメルマガを始めるなら、最初から高い配信サービスを契約しなくてもいいのでは。

そんな考えで、今回は **MailPoet × Brevo** の組み合わせを試しました。

MailPoetはWordPressの中でメルマガを作ったり、購読者を管理したりできるプラグイン。Brevoはメール配信を担当してくれるサービスです。

役割でいうと、こんな感じです。

- MailPoet：メルマガを書く、登録フォームを作る、購読者を管理する
- Brevo：SMTPでメールを送る、配信の安定性を支える

WordPressだけで全部やろうとすると、メールが迷惑メールに入りやすかったり、そもそも届かなかったりすることがあります。なので、送信部分だけは配信専門サービスに任せる。これが今回の狙いでした。

## うまくいくと思ったら、SMTPで止まった

Brevoのアカウントを作り、ドメイン認証を進めて、SMTPキーも取得。

ここまでは順調でした。

ところが、MailPoet側でSMTP設定をしてテスト送信しようとしたところ、

**Could not connect to SMTP host**

というエラー。

設定値を見直しても、ホスト名もログインもSMTPキーも合っている。DNSも大きく間違っていない。

こういうとき、だいたい「何か1つだけ小さいところが違う」んですよね。

## 587ではなく465で通った

Brevoの画面では、SMTPポートとして587が案内されることがあります。

ただ、今回のエックスサーバー上のWordPress環境では、587だと接続できませんでした。

そこで、ポートを **465** に変更し、安全な接続を **SSL/TLS** にしたところ、テスト送信が成功。

拍子抜けするくらい、すっと通りました。

同じようにMailPoetとBrevoをつないでいて、

- SMTPホストは合っている
- SMTPキーも合っている
- Brevoの専用ログインも入れている
- でも接続できない

という場合は、まずポート465を試してみる価値があります。

## DNS認証も地味に大事

SMTP接続だけ通れば終わり、ではありません。

メールをちゃんと届けるには、Brevo側のドメイン認証も大事です。

Brevoでドメインを追加すると、DNSに入れるCNAMEやTXTレコードが表示されます。それをサーバー側のDNS設定に追加していきます。

ここで注意したいのが、DMARCです。

すでに自分のドメインでDMARCを設定している場合、Brevoが提案するDMARCレコードをそのまま追加すると、DMARCが2つになってしまうことがあります。

DMARCは1ドメインに1つだけ。

ここを二重にすると、かえって認証が壊れます。

## 無料で始められるけど、雑にはできない

MailPoetとBrevoの組み合わせは、かなり現実的です。

小規模なサイトなら、Brevoの無料枠である1日300通でも十分に始められます。

ただし、メール配信は「送信ボタンを押せば届く」ものではなくなっています。

送信者アドレス、SMTP設定、ドメイン認証、配信解除リンク。

このあたりをきちんと整えておくことで、ようやく安心して使える仕組みになります。

## 今回の学び

- WordPressでメルマガを始めるなら、MailPoet × Brevo は選択肢になる
- BrevoのSMTP接続で587が通らない場合、465 + SSL/TLSを試す
- SMTPログインは自分のメールアドレスではなく、Brevo発行の専用ログインを使う
- DMARCは二重登録しない
- メルマガは本文より前に「届く設定」が大事
