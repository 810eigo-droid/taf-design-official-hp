# SPF / DKIM / DMARC 設定手順書(エックスサーバー × taf-design.com)

> 作成: 2026-07-22 / 対象: taf-design.com(エックスサーバー・WordPress/Lightning)
> 目的: メルマガ(MailPoet予定)だけでなく、CF7の送信通知・自動返信メールの到達率を底上げする
> 費用: **0円**(すべてエックスサーバーの標準機能)/ 作業時間目安: **15〜30分**

---

## 0. 全体像 — 3つの技術は「本人確認の3点セット」

```
                        受信側(Gmailなど)のチェック
  ┌──────────────────────────────────────────────────┐
  │                                                  │
  │  SPF   「このサーバーから送っていいって           │
  │         DNSに書いてある?」…… 送信元サーバーの確認  │
  │                                                  │
  │  DKIM  「メールに付いた電子署名は                 │
  │         本物か? 改ざんされてないか?」…… 署名の確認 │
  │                                                  │
  │  DMARC 「SPF/DKIMに失敗したメールを               │
  │         どう扱う? 報告はどこへ?」…… ポリシー宣言   │
  │                                                  │
  └──────────────────────────────────────────────────┘
```

- **SPF** = 「うちのドメインのメールは、このサーバーから送ります」という宣言(DNSのTXTレコード)
- **DKIM** = メール1通ごとに付く電子署名。公開鍵をDNSに置き、受信側が照合する
- **DMARC** = 「SPF/DKIMに落ちたメールは受け取らないで/隔離して/何もしないで」+「結果を私にレポートして」という宣言

2024年以降、**Gmailはこれらが未設定のメールを受け取り拒否・迷惑メール行きにする方針を強化**しており、
リード獲得フォーム(offer2)の自動返信が迷惑メールに落ちる=機会損失に直結する。設定は実質必須。

> 💡 この3点セットの設定スキルは、制作クライアント向けの商品
> (「お問い合わせフォームのメールが届かない」問題の解決)としてもそのまま使える。

## 1. 作業の流れ(チェックリスト)

| # | 作業 | 場所 | 所要 |
|---|---|---|---|
| ☐ 1 | DKIMをONにする | サーバーパネル > DKIM設定 | 5分 |
| ☐ 2 | SPFレコードの存在確認 | サーバーパネル > DNSレコード設定 | 5分 |
| ☐ 3 | DMARCを設定(まず p=none) | サーバーパネル > DMARC設定 | 5分 |
| ☐ 4 | テスト送信して PASS を確認 | Gmail / mail-tester.com | 10分 |
| ☐ 5 | (翌日以降)DMARCレポートを眺める | 通知メールアドレス | — |

**順番が大事**: DMARCはSPF/DKIMの結果を使う仕組みなので、必ず 1 → 2 → 3 の順で。
特にDKIMが未設定のままDMARCを厳しくすると、自分のメールが届かなくなるリスクがある。

## 2. STEP 1: DKIM設定(サーバーパネルでONにするだけ)

### 手順

```
サーバーパネル(https://secure.xserver.ne.jp/xapanel/login/xserver/server/)
  └─ ログイン
      └─ カテゴリ「メール」内の【DKIM設定】をクリック
          └─ ドメイン一覧から「taf-design.com」の【選択する】
              └─ 「DKIM設定一覧」タブ
                  └─ 設定状況が【OFF】になっていたらクリック → 【ON】に切り替え
                      └─ 完了(確認画面が出たら「設定する」)
```

### ポイント

- taf-design.com は**エックスサーバーのネームサーバーを利用中なので、DKIM用のDNSレコード
  (公開鍵)は自動で追加される**。手作業でのレコード追加は不要。
- 追加されたレコードの中身は「DKIM設定一覧」の**[表示]ボタン**で確認できる
  (`セレクタ名._domainkey.taf-design.com` という名前のTXTレコード)。
- もし将来、他社DNS(Cloudflareなど)に移した場合は、この[表示]で出てくるレコードを
  移転先DNSに手動で追加する必要がある。
- 反映まで**最大24時間**(通常は数分〜1時間程度)。

## 3. STEP 2: SPFレコードの確認(たぶん既にある。無ければ追加)

### 手順

```
サーバーパネル
  └─ カテゴリ「ドメイン」内の【DNSレコード設定】をクリック
      └─ 「taf-design.com」の【選択する】
          └─ 「DNSレコード一覧」タブ
              └─ 種別【TXT】で、内容が「v=spf1」で始まる行を探す
```

### 判定

| 状態 | 対応 |
|---|---|
| `v=spf1 ... ~all` の行が**ある** | ✅ 何もしなくてOK(エックスサーバーは2019年以降の契約でデフォルト設定済み) |
| **ない** | 「DNSレコード追加」タブから下記を追加 |

無い場合に追加する値(エックスサーバー標準):

| 項目 | 値 |
|---|---|
| ホスト名 | (空欄のまま = taf-design.com 自体) |
| 種別 | TXT |
| 内容 | `v=spf1 +a:svXXXXX.xserver.jp +a:taf-design.com +mx ~all` |

`svXXXXX.xserver.jp` は自分のサーバー番号(サーバーパネルのトップ「サーバー情報」の
ホスト名に書いてある)。

### ⚠️ 最重要の注意

- **SPFレコード(v=spf1のTXT)は1ドメインに1つだけ**。2つあると逆に認証エラーになる。
  既にあるのに追加しようとすると「同一『ホスト名』にSPFレコードは複数登録できません」と
  エラーが出る。その場合は**既存の行を編集**する。
- 将来、Brevo等の外部送信サービスを使うときも「新規追加」ではなく
  **既存のv=spf1の行に `include:` を書き足す**(例: `v=spf1 +a:svXXXXX.xserver.jp +mx include:spf.brevo.com ~all`)。

## 4. STEP 3: DMARC設定(まずは p=none で「観測」から)

エックスサーバーはサーバーパネルに**DMARC設定機能が標準搭載**されている
(DNSレコードを手で書かなくてよい)。

### 手順

```
サーバーパネル
  └─ カテゴリ「メール」内の【DMARC設定】をクリック
      └─ 「taf-design.com」の右端【編集】(または選択する)
          └─ 各項目を設定 → 【設定する】をクリック
```

### 最初に入れる値(観測モード)

| 項目 | 設定値 | 意味 |
|---|---|---|
| DMARCポリシー | **「何もしない」(= p=none)** | 認証に失敗しても配送はブロックしない。まず現状把握 |
| レポート設定 | **ON** | 受信側(Gmail等)から集計レポートを受け取る |
| 通知メールアドレス | `dmarc@taf-design.com` など自ドメインのアドレス | レポートの送り先 |

> 📮 通知メールアドレスは、サーバーパネル >「メールアカウント設定」で
> `dmarc@taf-design.com` を新規作成し、**Gmail(普段見るアドレス)へ転送設定**しておくと楽。
> ※ Gmailアドレスを直接指定すると、外部ドメイン宛レポートは受信側に無視される場合があるため
> 自ドメインのアドレスを推奨。

### (参考)手動でDNSレコードを書く場合

DMARC設定機能を使わない/他社DNSの場合は、DNSレコード設定に以下を追加するのと同じこと:

| 項目 | 値 |
|---|---|
| ホスト名 | `_dmarc` |
| 種別 | TXT |
| 内容 | `v=DMARC1; p=none; rua=mailto:dmarc@taf-design.com` |

### 段階的な強化ロードマップ(いきなり reject にしない)

```
p=none ──(1〜2ヶ月レポート観測、正規メールが全部PASSしてるか確認)──▶
p=quarantine ──(さらに1ヶ月問題なければ)──▶ p=reject
「何もしない」        「迷惑メールフォルダへ」            「受信拒否」
```

- 急に `reject` にすると、設定漏れのある正規メール(例: 外部サービス経由の送信)まで
  届かなくなるリスクがある。**none → quarantine → reject の順で**。
- MailPoet/Brevo導入(タスク3・4)の前後は、レポートで新しい送信経路がPASSしているか要確認。

## 5. STEP 4: 設定できたか確認する(3つの方法)

### 方法A: Gmailに送って「メッセージのソース」を見る(いちばん確実)

1. offer2フォームからテスト送信する(自動返信がGmail宛に届くようにする)。
   もしくはWebメール/メールソフトから `xxx@taf-design.com` → 自分のGmailへ送信
2. Gmailで届いたメールを開く → 右上の「︙」→ **「メッセージのソースを表示」**
3. 上部にこう出ていれば合格:

```
SPF:   PASS(IP: xxx.xxx.xxx.xxx)
DKIM:  'PASS'(ドメイン: taf-design.com)
DMARC: 'PASS'
```

※ 迷惑メールフォルダに入っていないかも同時に確認。

### 方法B: mail-tester.com(総合採点・無料)

1. https://www.mail-tester.com/ を開くと使い捨てアドレスが表示される
2. そのアドレス宛に taf-design.com からメールを送る
3. 「Then check your score」→ 10点満点で採点。SPF/DKIM/DMARCの合否が個別に出る
   (目標: 9〜10点)

### 方法C: コマンドで直接DNSを確認(制作クライアント対応時にも使える)

```bash
# SPF
nslookup -type=TXT taf-design.com
# → v=spf1 ... ~all が1つだけ出ればOK

# DMARC
nslookup -type=TXT _dmarc.taf-design.com
# → v=DMARC1; p=none; ... が出ればOK

# DKIM(セレクタ名はDKIM設定一覧の[表示]で確認したもの)
nslookup -type=TXT <セレクタ名>._domainkey.taf-design.com
# → v=DKIM1; k=rsa; p=MIIB... と長い公開鍵が出ればOK
```

Webで確認するなら MXToolbox(https://mxtoolbox.com/SuperTool.aspx)の
「SPF Record Lookup」「DMARC Lookup」「DKIM Lookup」でも同じことができる。

## 6. よくあるつまずき(FAQ)

| 症状 | 原因と対処 |
|---|---|
| 設定直後にテストしたらFAIL | DNS反映待ち。数時間〜最大24時間おいて再テスト |
| SPFがFAILする | v=spf1のTXTが2行ある(→1行に統合)/WP Mail SMTPの送信経路がSPFに含まれていない |
| DKIMがPASSしない | DKIM設定がOFFのまま/反映待ち/送信元がエックスサーバー以外(外部SMTP)になっている |
| DMARCだけFAIL | Fromのドメインと、SPF/DKIMで認証されたドメインが不一致(アライメント)。CF7の「送信元」を `wordpress@taf-design.com` 等の自ドメインにする |
| CF7の通知が急に迷惑メール行き | Gmail側のドメイン評価が原因のことも。次タスクのGoogle Postmaster Toolsで監視 |

### 現在の構成(WP Mail SMTP)との関係

- 現状、CF7の通知・自動返信は **WP Mail SMTP経由**。送信経路がエックスサーバーの
  SMTP(`svXXXXX.xserver.jp`)になっていれば、今回の設定がそのまま全部効く。
- WP Mail SMTPの設定画面で「差出人メールアドレス」が **@taf-design.com** に
  なっていることを確認(gmail.comなどを差出人にするとDMARCアライメントで落ちる)。
- 将来Brevoを接続したら(タスク3)、**Brevo側で発行されるDKIMレコードの追加**と
  **SPFへの `include:spf.brevo.com` 追記**が別途必要になる(このファイルに追記予定)。
